# Branch Live — Invoice Template Specification

## Invoice Design

All invoices are auto-generated by Stripe Invoicing and rendered as branded PDFs. Customers receive the PDF via email and can access all invoices from the Branch Live dashboard under **Billing → Invoice History**.

## Required Invoice Fields (for compliance & clarity)

| # | Field | Source | Notes |
|---|-------|--------|-------|
| 1 | Invoice number | Auto-increment: `BL-YYYY-NNNNNN` | Sequential, unique |
| 2 | Issue date | `invoice.created` timestamp | |
| 3 | Due date | Same as issue date (auto-charged) | |
| 4 | Business name & address | Stripe account settings | Branch Live LLC |
| 5 | Customer name & address | Stripe Customer object | |
| 6 | Customer account # | `accounts.id` from branchlive.db | |
| 7 | Line item descriptions | Stripe Price nickname | Include date range |
| 8 | Quantity × Unit price | Stripe Invoice Items | |
| 9 | Subtotal | Sum of line items | Pre-tax |
| 10 | Tax amount | Stripe Tax calculation | By state/country |
| 11 | Total | Subtotal + Tax | |
| 12 | Amount paid | Stripe charge amount | |
| 13 | Payment method | Last 4 digits + brand | `payment_method.card.last4` |
| 14 | Payment date | Charge timestamp | |
| 15 | Service period | `current_period_start` → `current_period_end` | |
| 16 | Payment terms | Static text | Auto-charge, cancel policy |
| 17 | Support contact | Static text | billing@branchlive.ai |
| 18 | EIN / Tax ID | Static text | For business customers |

## Invoice Special Cases

### Prorated Invoices (Mid-cycle Upgrades)
When a customer adds an upsell mid-cycle:
```
DESCRIPTION                                  QTY   RATE      AMOUNT
──────────────────────────────────────────────────────────────────
Web Add-on (full month: July 23 – Aug 22)   1     $10.00    $10.00
Credit: Unused portion of current period           -$3.33   -$3.33
(July 23 – Aug 2 = 11 days remaining)
                                               ─────────────
                                               CHARGE:      $6.67
```

### Credit Notes (Service Issues)
If a service credit is issued (e.g., outage compensation):
- Listed as negative line item with reason
- Example: `Service Credit — July 15 outage (2 hours)   -$5.00`

### Final Invoice (Cancellation)
See `05-cancellation-flow.md` for details. Key differences:
- Marked "FINAL INVOICE" prominently
- No future billing date
- Statement: "Your subscription has been canceled. No further charges will be made."
- Data retention notice: "Your call data will be retained until [date + 90 days]."

## Stripe Configuration

```yaml
# Stripe Dashboard Settings
invoice:
  branding:
    logo: "branchlive-logo.png"       # 512x512, uploaded to Stripe
    primary_color: "#0d1117"          # Branch Live dark theme
    accent_color: "#58a6ff"           # Branch Live accent
  footer: |
    Branch Live — AI Receptionist Service
    Questions? billing@branchlive.ai
    Cancel anytime from your dashboard.
  auto_advance: true                  # Auto-finalize drafts
  collection_method: charge_automatically
  days_until_due: null                # Charged immediately
  custom_fields:
    - name: "Account Number"
      value: "{{metadata.account_id}}"
```

## Pro-Forma Invoice (for Sales)

When a prospect requests a quote before signing up:

- Same template but:
  - Marked **"QUOTE — Not a Bill"**
  - No invoice number (placeholder)
  - "Valid until: [date + 14 days]"
  - Call-to-action: "Start your free trial at app.branchlive.ai/signup"
